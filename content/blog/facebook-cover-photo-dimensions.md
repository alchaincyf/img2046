## Facebook个人主页封面照片尺寸详解

### 标准尺寸和安全区域

**推荐尺寸**: 820 x 312 像素
**最小尺寸**: 400 x 150 像素
**显示尺寸**:
- 桌面端: 820 x 312 像素
- 移动端: 640 x 360 像素（左右各裁剪约90像素）
**安全区域**: 中心核心内容区域（约688 x 188 像素）
**文件格式**: JPEG或PNG
**最大文件大小**: 100KB（建议压缩优化）

### 桌面端和移动端显示差异

**桌面端显示** (820 x 312):
```
完整尺寸: 820 x 312
┌────┬──────────────────┬────┐
│裁剪│   核心显示区     │裁剪│
│区域│   (688x188)      │区域│
│ 约 │                  │ 约 │
│66px│   安全区域       │66px│
└────┴──────────────────┴────┘
```

**移动端显示** (640 x 360):
```
移动端只显示中间 640px 宽度
左右各约 90px 在移动端被裁剪
重要内容必须放在中心 688px 范围内
```

### 封面照片设计最佳实践

**内容布局策略**:
1. **核心内容放在中心安全区** (688 x 188 像素)
   - 品牌名称或Logo
   - Slogan或核心信息
   - 联系方式（网站、电话）
   - 行动号召（如"立即购买"）

2. **边缘区域设计**:
   - 使用装饰性图形
   - 渐变或纯色背景
   - 可接受被裁剪的元素
   - 延伸性图案（如云朵、波浪）

3. **头像重叠区域** (左下角):
   - 头像会覆盖封面左下角约 160 x 160 像素区域
   - 避免在这里放置重要文字或Logo
   - 可以在这里使用渐变或装饰元素

**设计技巧**:
- 使用高分辨率图片（至少1640 x 624 像素），然后缩小
- 避免纯文字设计，图片更吸引人
- 使用品牌色调保持一致性
- 可以添加二维码（放在中心右侧）
- 保持简洁，避免信息过载

**常见错误**:
- ❌ 重要Logo或文字在左下角（被头像遮挡）
- ❌ 联系方式在边缘区域（移动端被裁剪）
- ❌ 图片分辨率过低导致模糊
- ❌ 颜色与头像冲突，难以区分
- ❌ 文字过小，在移动端不可读

### 封面照片更换策略

**更换频率建议**:
- **品牌宣传**: 季节性更换（每3个月）
- **活动推广**: 活动前2周更换，活动结束后恢复
- **产品发布**: 新品发布期间更换
- **节日促销**: 节日期间更换特殊版本

**A/B测试策略**:
- 准备2-3个封面版本
- 使用Facebook A/B测试功能
- 测试7-14天
- 分析点击率和转化率
- 选择最佳版本长期使用

## Facebook个人资料图片（头像）尺寸

### 标准尺寸规范

**推荐尺寸**: 180 x 180 像素
**最小尺寸**: 100 x 100 像素
**显示尺寸**（会因位置而变化）:
- 个人主页: 140 x 140 像素（圆形）
- 在帖子旁边: 40 x 40 像素（圆形）
- 在评论中: 32 x 32 像素（圆形）
- 在消息列表: 50 x 50 像素（圆形）
- 在搜索结果: 44 x 44 像素（圆形）

**文件格式**: JPEG或PNG
**文件大小**: 建议<50KB

### 头像设计最佳实践

**个人账号头像**:
- 使用清晰的专业头像照
- 面部占据画面60-70%
- 表情亲切自然，眼神接触镜头
- 背景简洁不抢眼（纯色最佳）
- 光线均匀，避免过度阴影
- 穿着与个人定位匹配

**品牌/企业头像**:
- 使用简洁的品牌Logo
- 避免过多细节和文字
- 确保在40 x 40像素下仍能识别
- 使用品牌主色调
- 可以使用品牌首字母作为替代
- 保持与封面照片风格一致

**测试头像质量**:
在多个尺寸下测试头像：
- [ ] 在180 x 180像素（原始尺寸）清晰
- [ ] 在140 x 140像素（主页）可辨认
- [ ] 在40 x 40像素（帖子）仍能识别
- [ ] 圆形裁剪后不损失重要内容
- [ ] 在浅色和深色背景下都清晰
- [ ] 与封面照片搭配和谐

**头像优化技巧**:
1. **高分辨率原图**: 至少500 x 500像素
2. **居中构图**: 重要内容在中心
3. **圆形裁剪预览**: 设计时就考虑圆形显示
4. **高对比度**: 确保在小尺寸下清晰
5. **避免细节**: 小尺寸下细节会丢失
6. **保持简洁**: 少即是多

### 头像与封面照片的搭配

**视觉连贯性**:
- 使用相同的色调方案
- 保持风格一致（专业、活泼、极简等）
- 头像可以在封面中找到对应元素
- 统一的视觉语言

**色彩搭配技巧**:
- 封面使用头像的主色调作为背景
- 或使用互补色增加对比
- 避免颜色冲突（如都是红色）
- 测试在不同设备上的显示效果

## Facebook帖子图片尺寸

Facebook支持多种帖子图片比例，根据内容类型选择合适的尺寸。

### 正方形帖子图片

**推荐尺寸**: 1200 x 1200 像素
**比例**: 1:1（正方形）
**最大文件**: 30MB
**格式**: JPEG或PNG

**适用场景**:
- 产品展示
- 人物肖像
- 信息图表
- 文字卡片
- 通用内容

**优势**:
- 在桌面端和移动端都占据较大空间
- 与其他内容形成鲜明对比
- 在信息流中更引人注目
- 适合Instagram跨平台发布

### 横版帖子图片

**推荐尺寸**: 1200 x 630 像素
**比例**: 1.91:1（横版）
**最大文件**: 30MB

**适用场景**:
- 链接分享缩略图（最重要）
- 风景摄影
- 团体合影
- 横构图内容
- 宽场景展示

**链接分享优化**:
当分享网站链接时，Facebook会自动抓取缩略图：
- 确保1200 x 630像素的OG图像
- 重要内容在中心区域（可能被裁剪）
- 使用清晰的标题或品牌Logo
- 高对比度，吸引点击

**OG标签设置**:
```html
<meta property="og:image" content="https://example.com/image.jpg" />
<meta property="og:image:width" content="1200" />
<meta property="og:image:height" content="630" />
```

### 竖版帖子图片

**推荐尺寸**: 1200 x 1500 像素（最大1200 x 1890）
**比例**: 4:5（竖版）
**最大文件**: 30MB

**适用场景**:
- 时装展示
- 人物全身照
- 产品竖构图
- 信息图表
- 需要更多垂直空间的内容

**优势**:
- 在移动端占据更大屏幕空间
- 研究显示竖版互动率比横版高
- 更适合手机浏览习惯
- 在信息流中更突出

**注意事项**:
- 高度不要超过1890像素（会被裁剪）
- 重要内容在中心区域
- 底部预留空间（可能被文字说明遮挡）

## 常见问题解答

### 1. Facebook封面照片在手机上显示不全怎么办？

**问题原因**: Facebook在移动端会裁剪封面照片左右各约90像素。

**解决方法**:

- 重要内容（Logo、文字、联系方式）放在中心688像素宽度内

**预防措施**:
- 设计时就考虑移动端显示
- 使用"安全区域"网格辅助设计
