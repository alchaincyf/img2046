## 图片压缩原理

### 有损压缩 vs 无损压缩

**1. 无损压缩（Lossless）**

**原理**：
- 通过算法重新组织数据
- 去除冗余信息
- **不丢失任何图像数据**
- 可以完全还原

**压缩率**：
- 文件大小减少：**30-50%**
- 画质：**完全无损**

**适用格式**：
- PNG（主要）
- GIF
- WebP（无损模式）

**优点**：
```
✅ 画质完全无损
✅ 适合需要多次编辑的图片
✅ 保留透明通道
✅ 适合图标、logo
```

**缺点**：
```
❌ 压缩率有限
❌ 大尺寸照片效果不明显
❌ 文件仍然较大
```

**2. 有损压缩（Lossy）**

**原理**：
- 删除人眼不敏感的细节
- 减少颜色数量
- **丢弃部分图像数据**
- 无法完全还原

**压缩率**：
- 文件大小减少：**60-90%**
- 画质：轻微到明显损失

**适用格式**：
- JPEG（主要）
- WebP（有损模式）
- HEIC

**优点**：
```
✅ 压缩率非常高
✅ 文件大幅减小
✅ 适合照片、复杂图像
✅ 视觉上可能无明显差异
```

**缺点**：
```
❌ 画质有损失
❌ 反复压缩损失累积
❌ 可能出现压缩伪影
❌ 不适合多次编辑
```

**压缩伪影示例**：
```
过度压缩的JPEG会出现：
- 蚊子噪点（边缘噪点）
- 色块现象（平滑区域出现方块）
- 边缘模糊（细节丢失）
- 色带（渐变不连续）
```

### 色彩采样（Chroma Subsampling）

**JPEG压缩的重要参数**：

**什么是色彩采样？**
- 人眼对亮度（明暗）敏感，对色度（颜色）不敏感
- 可以减少色度信息，保持亮度信息
- 大幅减小文件大小，视觉影响小

**三种采样模式**：

**1. 4:4:4（无采样）**
```
保留完整色彩信息
- 画质：最佳
- 文件大小：最大
- 使用场景：专业摄影、需要后期处理
```

**2. 4:2:0（标准采样）**
```
色度分辨率为亮度的1/4
- 画质：良好
- 文件大小：减少30-40%
- 使用场景：网页、社交媒体（推荐）
```

**3. 4:2:2（中等采样）**
```
色度分辨率为亮度的1/2
- 画质：介于两者之间
- 文件大小：减少15-25%
- 使用场景：高质量网页
```

**实际效果**：
```
4:4:4 vs 4:2:0：
- 文件大小差异：约30-40%
- 视觉差异：正常观看几乎无差异
- 放大200%：轻微差异
- 推荐：4:2:0（网页使用）
```

```
问题：
- 每次压缩损失累积
- 画质逐渐下降
- 最终出现明显伪影

示例：
原始 → 压缩80% → 重新压缩80% → 再次压缩
质量：100% → 95% → 85% → 70%（肉眼可见）
```

**最佳实践**：
```
✅ 保留原始高质量文件
✅ 从原始文件重新导出不同质量
✅ 仅在最终输出时压缩
❌ 避免对已压缩文件再次压缩
❌ 不要反复打开编辑JPEG
```

**无损压缩（PNG）**：
```
可以反复压缩：
- 不损失画质
- 但压缩率递减
- 第一次效果最明显
```

**工作流程建议**：
```
专业工作流：
1. 保存原始文件（RAW/TIFF/高质量JPEG）
2. 编辑原始文件
3. 根据需求导出不同压缩版本
4. 保留原始文件供将来使用

个人工作流：
1. 保留手机/相机原始照片
2. 需要时从原始压缩
3. 不要覆盖原始文件
```
