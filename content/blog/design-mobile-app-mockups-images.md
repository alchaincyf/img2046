# 创建App界面原型图:设计移动应用的图片资源

移动应用设计是一个复杂的过程,需要为各种屏幕尺寸和分辨率准备不同的图片资源。作为设计师或开发者,你需要了解iOS和Android平台的规范,掌握@2x和@3x的含义,以及如何高效地创建和管理这些设计资源。本文将详细介绍使用图像魔方创建App界面原型图的完整流程。

## 为什么需要了解App设计规范?

### 多样化的设备生态系统

**iOS设备**:
- iPhone SE (375×667pt)
- iPhone 13/14 (390×844pt)
- iPhone 14 Pro Max (430×932pt)
- iPad Mini (744×1133pt)
- iPad Pro (1024×1366pt)

**Android设备**:
- 小屏手机 (360×640dp)
- 标准手机 (360×800dp)
- 大屏手机 (411×873dp)
- 平板 (600×960dp)

每种设备都有不同的屏幕尺寸、像素密度和显示特性。如果你的应用需要在所有设备上提供一致的视觉体验,就需要为每种情况准备相应的图片资源。

### 设计资源的类型

移动应用通常需要以下类型的图片资源:

1. **界面截图** - 用于App Store、Google Play展示
2. **图标** - App图标、功能图标、Tab图标
3. **启动画面** - Launch Screen、Splash Screen
4. **背景图** - 首页背景、引导页背景
5. **插图** - 空状态、功能说明插画
6. **营销图片** - 推广图、Banner图

## iOS平台设计规范

### 像素与点的概念

**Point (pt)** - 逻辑像素
**Pixel (px)** - 物理像素

iOS使用点作为布局单位,但实际渲染时转换为物理像素。

**公式**:
```
物理像素 = 逻辑像素 × 缩放因子
```

**示例**:
```
iPhone 14 Pro:
- 逻辑尺寸: 393×852pt
- 物理尺寸: 1179×2556px
- 缩放因子: @3x
```

### @2x和@3x的含义

**@2x** - 2倍分辨率
- 用于iPhone 6/7/8、iPhone X系列
- 逻辑尺寸: 375×667pt
- 物理尺寸: 750×1334px

**@3x** - 3倍分辨率
- 用于iPhone Plus系列、iPhone 12/13/14
- 逻辑尺寸: 414×896pt
- 物理尺寸: 1242×2688px

**命名规范**:

```
icon.png          (1x, 基础尺寸)
icon@2x.png       (2x, 2倍尺寸)
icon@3x.png       (3x, 3倍尺寸)
```

**实例**:

设计一个44×44pt的按钮图标:
```
icon.png         44×44px   (旧款设备)
icon@2x.png      88×88px   (iPhone 8)
icon@3x.png      132×132px (iPhone 14)
```

### iOS标准尺寸

**常见界面元素尺寸**:

| 元素 | 尺寸(pt) | @2x (px) | @3x (px) |
|------|---------|----------|----------|
| 状态栏高度 | 44 | 88 | 132 |
| 导航栏高度 | 44 | 88 | 132 |
| Tab栏高度 | 49 | 98 | 147 |
| 按钮最小点击区域 | 44×44 | 88×88 | 132×132 |
| 图标尺寸 | 24×24 | 48×48 | 72×72 |
| 大图标 | 64×64 | 128×128 | 192×192 |

**App图标尺寸**:

| 用途 | 尺寸 (px) |
|------|----------|
| iPhone App (@2x) | 120×120 |
| iPhone App (@3x) | 180×180 |
| iPad Pro (@2x) | 167×167 |
| App Store | 1024×1024 |

### 使用图像魔方创建iOS资源

**步骤1:设计基础版本 (@1x)**

1. 打开图像魔方自由画布
2. 创建新画布,设置尺寸为375×667px (iPhone 8基准)
3. 设计界面原型

**步骤2:导出@2x版本**

1. 框选设计区域
2. 点击"导出"
3. 设置缩放比例为200%
4. 导出为PNG
5. 文件名添加@2x后缀

**步骤3:导出@3x版本**

1. 重复上述步骤
2. 设置缩放比例为300%
3. 文件名添加@3x后缀

**步骤4:切片导出**

如果界面中有多个图标:

1. 使用图像魔方的"切片"功能
2. 框选每个图标
3. 批量导出为@2x和@3x
4. 自动添加正确的后缀

## Android平台设计规范

### dp和sp单位

**dp (Density-independent Pixel)** - 密度无关像素
- 用于布局尺寸
- 自动适应不同屏幕密度

**sp (Scale-independent Pixel)** - 缩放无关像素
- 用于字体大小
- 根据用户系统设置调整

**换算关系**:

```
1 dp = 1 px (mdpi, 基准密度)
1 dp = 1.5 px (hdpi)
1 dp = 2 px (xhdpi)
1 dp = 3 px (xxhdpi)
1 dp = 4 px (xxxhdpi)
```

### 密度桶(Density Buckets)

Android将设备分为多个密度桶:

| 密度桶 | 倍数 | 设备示例 | 资源文件夹 |
|--------|------|----------|-----------|
| mdpi | 1x | 基准 | drawable-mdpi/ |
| hdpi | 1.5x | 低端手机 | drawable-hdpi/ |
| xhdpi | 2x | 中端手机 | drawable-xhdpi/ |
| xxhdpi | 3x | 高端手机 | drawable-xxhdpi/ |
| xxxhdpi | 4x | 超高清 | drawable-xxxhdpi/ |

**标准设计基准**: xxhdpi (3x)

### Android标准尺寸

**常见界面元素尺寸**:

| 元素 | 尺寸 (dp) | xxhdpi (px) |
|------|----------|-------------|
| 状态栏高度 | 24 | 72 |
| 工具栏高度 | 56 | 168 |
| 按钮最小高度 | 48 | 144 |
| 点击区域最小 | 48×48 | 144×144 |
| 图标尺寸 | 24×24 | 72×72 |
| 导抽宽度 | 280 | 840 |

**App图标尺寸**:

| 用途 | 尺寸 (dp) | xxhdpi (px) |
|------|----------|-------------|
| 启动图标 | 48×48 | 144×144 |
| Google Play | 512×512 | 1536×1536 |
| 通知图标 | 24×24 | 72×72 |

### 使用图像魔方创建Android资源

**步骤1:设计xxhdpi版本 (3x)**

1. 创建画布,尺寸为360×640dp * 3 = 1080×1920px
2. 设计界面原型

**步骤2:导出其他密度版本**

1. 使用图像魔方的"批量导出"功能
2. 设置多个缩放比例:
   - xxxhdpi (4x) - 133%
   - xxhdpi (3x) - 100% (基准)
   - xhdpi (2x) - 67%
   - hdpi (1.5x) - 50%
   - mdpi (1x) - 33%
3. 导出到不同文件夹

**步骤3:重命名文件**

Android不需要文件名后缀,但需要放在正确的文件夹:
```
res/
  drawable-mdpi/icon.png
  drawable-hdpi/icon.png
  drawable-xhdpi/icon.png
  drawable-xxhdpi/icon.png
  drawable-xxxhdpi/icon.png
```

## 完整设计流程

### 阶段1:准备阶段

**1. 确定目标设备**

主要支持哪些设备?
- iPhone为主?
- Android为主?
- 跨平台?

**2. 选择设计基准**

**iOS**:
- iPhone 8 (375×667pt @2x)
- iPhone 14 (393×852pt @3x)

**Android**:
- 标准手机 (360×640dp @3x)
- 大屏手机 (411×873dp @3x)

**3. 准备设计规范**

创建设计系统文档:
- 色彩规范
- 字体规范
- 间距规范
- 组件规范

### 阶段2:设计阶段

**1. 使用图像魔方创建画布**

选择正确的画布尺寸:
```
iOS (iPhone 14):
- 逻辑尺寸: 393×852pt
- 设计尺寸 (@3x): 1179×2556px
```

**2. 设置参考线**

为关键布局添加参考线:
- 状态栏: 顶部44pt (132px @3x)
- 导航栏: 紧接状态栏,44pt高
- 内容区: 导航栏下方
- Tab栏: 底部49pt (147px @3x)

**3. 设计界面元素**

**导航栏设计**:
1. 绘制44pt高的矩形
2. 添加标题文字(17-20pt)
3. 左侧返回按钮(22×22pt图标)
4. 右侧操作按钮(22×22pt图标)

**列表项设计**:
1. 绘制容器,高度44-48pt
2. 左侧图标(24×24pt)
3. 标题文字(17pt,常规)
4. 副标题文字(15pt,灰色)
5. 右侧箭头(12×12pt)

**按钮设计**:
1. 绘制按钮背景,最小高度44pt
2. 添加圆角(4-8pt)
3. 按钮文字(17pt,中等)
4. 点击态颜色(透明度80%)

### 阶段3:适配阶段

**1. iOS适配**

**iPhone SE (小屏)**:
- 检查文字是否被截断
- 调整内边距
- 隐藏次要元素

**iPhone 14 Pro Max (大屏)**:
- 增加内容间距
- 添加更多装饰元素
- 利用额外空间

**iPad (平板)**:
- 使用两栏布局
- 增加内容密度
- 调整字体大小

**2. Android适配**

**小屏手机 (360dp)**:
- 减少内边距
- 使用紧凑字体
- 隐藏次要元素

**大屏手机 (411dp)**:
- 增加内边距
- 使用舒适字体
- 展示更多内容

**平板 (600dp+)**:
- 使用多栏布局
- 增加内容密度
- 侧边栏导航

### 阶段4:导出阶段

**1. 导出iOS资源**

**单个图标**:
```
文件名格式: icon@2x.png, icon@3x.png

使用图像魔方:
1. 框选图标
2. 导出 > 设置缩放200%
3. 保存为 icon@2x.png
4. 重复,设置缩放300%
5. 保存为 icon@3x.png
```

**完整界面**:
```
1. 导出为PDF(用于展示)
2. 导出为PNG@3x(用于开发)
3. 导出切片(用于图标)
```

**2. 导出Android资源**

**单个图标**:
```
使用图像魔方批量导出:
1. 框选图标
2. 批量导出 > 设置多个缩放:
   - 100% (xxhdpi)
   - 67% (xhdpi)
   - 50% (hdpi)
   - 33% (mdpi)
3. 导出到对应文件夹
```

**完整界面**:
```
1. 导出为PNG(开发用)
2. 导出为WebP(优化大小)
3. 导出九宫格(9-patch,可拉伸图片)
```

## 最佳实践

### 1. 使用矢量格式

对于图标和简单图形,优先使用SVG:
- 无限缩放
- 文件小
- 易于维护

**导出流程**:
1. 在图像魔方中设计
2. 导出为SVG
3. 开发时使用Vector Asset(Android)或PDF(iOS)

### 2. 建立设计系统

创建统一的设计规范:

**颜色系统**:
```
主色: #007AFF
辅助色: #5AC8FA
警告色: #FF3B30
成功色: #34C759
背景色: #F2F2F7
文字色: #000000
次要文字: #8E8E93
```

**字体系统**:
```
标题1: 28pt, 粗体
标题2: 24pt, 粗体
标题3: 20pt, 中等
正文: 17pt, 常规
辅助: 15pt, 常规

```

**间距系统**:
```
超大间距: 32pt
大间距: 24pt
标准间距: 16pt
小间距: 8pt
微小间距: 4pt
```

### 3. 命名规范

使用清晰的命名:

**iOS**:
```
图标: ic_[名称]@[倍数].png
例: ic_home@2x.png

图片: img_[名称]_[尺寸]@[倍数].png
例: img_hero_375x667@2x.png
```

**Android**:
```
图标: ic_[名称]
例: ic_home.png (放在对应drawable文件夹)

图片: img_[名称]_[描述]
例: img_hero_background.png
```

## 常见问题

### Q1: @2x和@3x图片可以混用吗?

可以,但不推荐。

**混用场景**:
- 某些图标只提供@2x,系统会自动缩放
- 性能敏感场景,只加载@2x

**最佳实践**:
- 为所有资源提供@2x和@3x
- 确保在所有设备上都清晰

### Q2: Android和iOS可以共用一套资源吗?

技术上可以,但不推荐。

**原因**:
- 设计风格不同(Material Design vs Human Interface)
- 尺寸规范不同
- 图标风格不同

**跨平台方案**:
- React Native / Flutter项目
- 使用同一套设计系统
- 导出两套尺寸资源

### Q3: 如何处理刘海屏?

**iOS**:
- 使用安全区域(Safe Area)
- 内容避开顶部刘海
- 底部避开Home指示器

**Android**:
- 使用WindowInsets
- 避开状态栏和导航栏

**设计时**:
- 预留顶部44pt安全区
- 底部34pt安全区
- 关键内容放在安全区内

### Q4: 设计尺寸应该选多大?

**推荐基准**:

**iOS**:
- iPhone 8: 375×667pt (@2x)
- iPhone 14: 393×852pt (@3x)

**Android**:
- 标准手机: 360×640dp (@3x)
- 大屏手机: 411×873dp (@3x)

**建议**:
- 从主流尺寸开始
- 验证小屏和大屏适配
- 提供多种尺寸资源

### Q5: 如何处理暗黑模式?

**方案1:两套图片**
```
icon-light.png (浅色模式)
icon-dark.png (暗黑模式)
```

**方案2:使用tint color**
- 提供单色图标
- 运行时应用颜色

**方案3:使用PDF/矢量**
- 开发时动态着色
- 只需一套资源
