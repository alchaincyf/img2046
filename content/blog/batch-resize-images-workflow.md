## 批量处理前的准备工作

### 第一步:明确需求和目标

在开始批量处理前,回答以下问题:

**1. 图片用途**:
- 网站使用? → 需要优化文件大小
- 打印输出? → 需要高分辨率
- 社交媒体? → 需要符合平台规范
- 电商展示? → 需要统一尺寸和比例

**2. 目标尺寸**:
- 固定尺寸(如800x800px)?
- 固定宽度(如1920px宽,高度自动)?
- 按百分比(如缩小到50%)?

**3. 质量要求**:
- 需要保持原始质量?
- 可以接受轻微压缩?
- 优先考虑文件大小?

**4. 输出格式**:
- 保持原格式?
- 统一转换为JPEG?
- 使用现代格式如WebP?

### 第二步:检查原图质量

批量处理前必须确保原图符合要求:

**分辨率检查**:
- 原图尺寸至少是目标尺寸的1.5倍
- 示例: 目标800x800px, 原图至少1200x1200px

**格式统一**:
- 尽量使用相同格式(如全部JPEG)
- 混合格式需要分别处理

**文件完整性**:
- 检查文件是否损坏
- 确认可以正常打开

**命名规范**:
- 文件名不含特殊字符
- 避免中文路径(可能导致错误)

**示例检查清单**:
```
✓ 原图尺寸: 3000x2000px (目标: 1920x1280px)
✓ 格式: JPEG
✓ 文件大小: 平均2.5MB
✓ 文件名: product-001.jpg 到 product-500.jpg
✓ 色彩空间: sRGB
```

### 第三步:整理文件结构

良好的文件组织提高效率:

**建议结构**:
```
项目文件夹/
├── 原图/              (保留,不修改)
│   ├── raw-001.jpg
│   ├── raw-002.jpg
│   └── ...
├── 处理中/            (批量处理)
│   └── [上传的文件]
├── 完成/              (处理结果)
│   ├── processed-001.jpg
│   ├── processed-002.jpg
│   └── ...
└── 备份/              (重要:备份原图)
    └── [原图副本]
```

**整理步骤**:
1. 创建项目文件夹
2. 备份原图到"备份"文件夹
3. 重命名文件(如果需要)
4. 分类文件(如果有多组)

### 第四步:测试参数设置

正式批量处理前,先用几张图片测试:

**测试流程**:
```
1. 选择3-5张代表性图片
   (不同内容、不同尺寸)

2. 应用计划使用的参数
   (如调整到1200px宽,质量85)

3. 检查处理结果
   - 质量是否满意?
   - 尺寸是否正确?
   - 文件大小是否合理?

4. 必要时调整参数
   - 重新测试
   - 直到效果满意

5. 记录最终参数
   - 用于批量处理
```

**测试检查项**:
- [ ] 清晰度符合预期
- [ ] 比例未变形
- [ ] 文件大小在范围
- [ ] 色彩无明显变化
- [ ] 特殊图片(如透明背景)正常


##### 2.2 质量设置

**JPEG质量**:
- **85-90**: 高质量,适合网站(推荐)
- **80-85**: 标准质量,文件更小
- **70-80**: 较低质量,节省空间
- **95+**: 几乎无损,文件较大

**建议**:
- 电商产品图: 85-90
- 网站内容图: 80-85
- 社交媒体: 85

**PNG压缩**:
- 选择压缩级别
- 无损质量
- 仅影响文件大小

##### 2.3 格式选择

**保持原格式**:
- 最简单,推荐
- 不改变格式
- 保持透明度(PNG)

**统一转JPEG**:
- 文件更小
- 广泛兼容
- 适合照片
- 注意:透明背景变黑/白

**统一转WebP**:
- 最小文件大小
- 现代浏览器支持
- 提供JPEG备份

**格式选择建议**:
```
电商产品图:
- 如需透明背景 → PNG
- 如无透明 → JPEG质量85

社交媒体:
- 全部转JPEG质量85

网站优化:
- 转WebP + JPEG备份
- 考虑提供AVIF

专业摄影:
- 保持原格式
- 质量设为90+
```

##### 2.4 高级选项

**插值算法**:
- **Lanczos**: 最佳质量(推荐)
- **双三次**: 标准质量,速度快
- **双线性**: 快速,质量稍降

**调整后锐化**:
- 放大时:启用,强度20-30%
- 缩小时:通常不需要
- 批量处理建议:关闭

**文件命名**:
- 保持原名
- 添加后缀(如"-800px")
- 或使用批处理工具重命名

### 技巧3:自动化验证

**质量检查脚本**:
```python
import os
from PIL import Image

def verify_images(folder, target_size):
    errors = []
    for filename in os.listdir(folder):
        if filename.endswith('.jpg'):
            img = Image.open(os.path.join(folder, filename))
            if img.size != target_size:
                errors.append(f"{filename}: {img.size}")

    return errors

# 使用
errors = verify_images('./processed', (1920, 1280))
print(f"发现{len(errors)}个错误")
```

## 常见问题和解决方案

### 问题1:处理过程中失败或中断

**原因**:
- 网络连接中断
- 浏览器标签页关闭
- 文件过大导致超时
- 电脑休眠

**解决方案**:
```
1. 预防措施:
   - 使用稳定的网络连接
   - 保持标签页打开
   - 单次处理不超过50张
   - 调整电脑电源设置

2. 恢复方法:
   - 记录已完成的文件
   - 从失败位置继续
   - 重新处理失败的文件

3. 分批策略:
   - 大批量分批处理
   - 每批30-40张
   - 降低单次失败风险
```

### 问题2:部分图片处理结果异常

**常见异常**:
- 某些图片比例变形
- 部分图片质量特别差
- 个别文件损坏

**排查方法**:
```
1. 检查原图
   - 尺寸是否异常
   - 是否损坏
   - 格式是否一致

2. 隔离异常文件
   - 单独处理异常文件
   - 检查是否原图问题

3. 分批处理
   - 将异常文件单独一批
   - 使用不同参数
```

### 问题3:文件大小仍然过大

**原因**:
- 质量设置过高
- 图片内容复杂
- 尺寸仍然较大

**解决方案**:
```
1. 降低质量设置
   - 从85降到80
   - 测试质量损失是否可接受

2. 进一步缩小尺寸
   - 如从1920px降到1600px
   - 视觉损失通常不明显

3. 转换格式
   - JPEG转WebP
   - 可减少30%文件大小

4. 使用压缩工具
   - 先调整大小
   - 再使用压缩工具
   - 两步优化
```

### 问题4:透明背景丢失

**原因**:
- JPEG格式不支持透明背景
- 透明背景被填充为黑色或白色

**解决方案**:
```
1. 使用PNG格式
   - 保持透明背景
   - 文件较大但功能完整

2. 转换前添加背景色
   - 在透明图层上添加白色/品牌色背景
   - 然后转换为JPEG

3. 批量处理时分离
   - 需要透明的用PNG
   - 不需要的用JPEG
   - 分两批处理
```

### 问题5:处理后色彩变化

**原因**:
- 色彩空间转换
- JPEG压缩导致色彩偏移
- 不同软件的色彩管理差异

**解决方案**:
```
1. 统一色彩空间
   - 所有图片使用sRGB
   - 避免使用Adobe RGB或ProPhoto RGB

2. 提高质量设置
   - 使用更高的质量(90+)
   - 减少压缩损失

3. 使用无损格式
   - PNG无损保存
   - 文件较大但无色彩损失

4. 验证显示设备
   - 在目标设备上预览
   - 确保显示一致
```
